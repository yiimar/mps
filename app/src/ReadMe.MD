# DDD

### Эванс, Э. (2004) Доменно-ориентированный дизайн. п. 70

DDD действительно требует только одного изолированного уровня — уровня домена.

«Слой предметной области» — это проявление модели предметной области и всех непосредственно связанных с ней элементов дизайна.

Просто потому, что непрактично смешивать предметную область с другими частями программы — модель предметной области должна быть отделена. Как мы решим изолировать другие части конструкции, полностью зависит от нас.
